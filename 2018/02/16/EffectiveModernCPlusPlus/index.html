Chapter1: Deducting Types

  • Universal reference: They exists and are not the same as lvalue reference or rvalue reference

Item 1: Understanding type deduction

  • During tempalte type deduction, arguments that are references are treated as non-references.

  • When deducing types for universial reference parameters, lvalue arguments get special treatments - both T and ParamType are deduced to be lvalue references.

  • During template type deduction, arguments that are array or function names decay to pointers, unclass they’re used to initialize references.

Item 2: Understanding auto deduction

  • 3 cases:
    • pointer or reference, but not a universal reference(const auto& a = x; )
    • a universal reference (auto&& uref1 = x;)
    • neither a pointer or reference (auto x = 27;)
  • special type deduction which is the only difference between auto and std::initializer_list
    • auto x3 = {27}; // type is std::initializer_list

      Item 3: Understand decltype

  • std::vector is special. Operator [] does not return a bool&. Instead, it returns a brand new object. So a decltype is needed here.

  • For lvalue expressions of type T other than names, decltype always reports a type of T&. Ex decltype(x) - > int but decpltype((x))-> int& where x = 4;

  • c++14 has support for decltype(auto) that it perfomrs the type deduction using the decltype rules

Item 4: Know how to view deduced types

  • When? editing the code -> IDE; during compilation(P32); at runtime(typeid().name())
  • typeid().name() is not reliable because it’s treating type as if it has been passed to a tempalte function as a by-value parameter
  • Boost.TypeIndex is your friend
