Learn You a Haskell for Great Good

Introduction

  • imperative programming language VS fuctional language
    • (C,C++,Java,Python,..) VS (Haskell, ML, OCaml)
  • referential transparency:
    • if a function is called twice with the same parameters, it’s guaranteed to return the same result.
  • statically typed
    • know the type at compile time
  • run Haskell from terminal
    • ghci
    • :set prompt "ghci> "
  • stop Haskell from terminal
    • just use Ctrl + Z
  • load functions in a haskell file
    • :l myfunctions

Starting Out

  • 5 * -3 would fail in Haskell
  • Boolean algebra
    • True, False, &&, ||, ==, /=(means not equal)
  • infix function
    • +, * are all this type
  • prefix funciton( put before the values)
    • succ
      • return the successor
    • div
      • div 92 10 or 92div10
  • function application:
    • definition
  • condition statement
    • else is mandatory in Haskell
    • if is an expression: a piece of code that returns a value
  • Baby’s first functions
    • create a function and store it in file name.hs.
    • open terminal, and type in :l name to import the function
  • function orders in Haskell do not matter. (like python)
  • if statement:
    • else is mandatory
    • ```has no special meaning in Haskell’s syntax. It’s used either to denote a strict version fo a function or a slightly modified version of a function or a variable
  • functions:
    • function name can’t begin with uppercase letters
    • function doesn’t take any parameters becomes definition
  • list:
    • homogeneous data structure, which means that it stores several elements of the same type
    • we use the keyword let to define a name right in GHCI. let a = 1 equals a = 1
    • list concatenation: [] ++ [], “hello” ++ “world”
    • watch out: Hasekll has to walk through the whole list on the lift side of ++. So if there is 5 million elements on the left…..
      • use : efficiently cancatenate two lists.
        • Ex: ` ‘A’: “SMALL CAT” -> “A SMALL CAT”
    • !!: get an element out of the list by index
      • Ex "Steve Buscemi" !! 6 -> ‘B’
    • basic operation on list
      • head: first element in a list, tail: result after chops off a list’s head, last: last element, init: everything except its last element
        • these operation cannot operate on empty list
      • null, length, reverse, take, drop, maximum, minimum, sum, product, elem(works as in in Python)
        - `drop 3 [1,2,3,4]` -> [4]
- `5 `elem` [1,2,3]` -> False
  • Texas ranges
    • [1..20]: a list containing elements from 1 to 20
    • [1,2..20]: [1,2,4,…,20]
    • be careful with floating points in ranges.
    • take 24 [13,26..] will give you the first 24 multiples of 13( or you can use [13,26..24*13])
    • cycle: takes a list and cycles it into an infinite list Ex take 10 (cycle [1,2,3])
    • repeat: takes an element and produces an infinite list of just that element
  • list comprehensionG
    • [x*2 | x<- [1..10]]: [2,4,6,8…20]
    • [x*2 | x <- [1..10], x*2 >= 12]: [12,14,16,18,20]
    • boomBangs xs = [ if x < 10 then "BOOM!" else "BANG!" | x <- xs, odd x]
      omBangs [7..13] : [“BOOM!”,”BOOM!”,”BANG!”,”BANG!”]
    • length' xs = sum[1 | _ <- xs]: our own length calculation function
      • _ means that we don’t care what we’ll draw from the list anyway so instead of writing a variable name that we’ll never use, we just write _.
    • listed comprehension
      • [ [ x | x <- xs, even x ] | xs <- xxs]
  • Tuples
    • a tuple can hold elements of different types, like python, we use parenthesis. Use tuples when you know in advance how many components some piece of data should have.
    • zip: zip [1,2,3,4,5] [5,5,5,5,5]: [(1,5),(2,5),(3,5),(4,5),(5,5)]

Types and Typeclasses

  • Static type system + type inference
  • :t input-data: you can get type
  • a: type variable, which means that it can be an type. Functions that has type variables are called polymorphic functions
  • typeclasses:
    • Everything before the => symbol is called a class constraint
    • :t (==): the equality function takes any two values that are of the same type and returns a Bool. The type of those two values must be a member of the Eq class (this was the class constraint).
    • type annotation: explicitly saying that the type of an expression should be
    • read "5" :: Int
    • Types: (), Bool, Char, Ordering, Int, Integer(unlimited number), Float, Double and Bounded
    • fromIntegral: useful to do type conversion. Ex: fromIntegral (length[1,2,3,4]) + 3.2. type of length is int, but you can not add an int with double. with fromIntegral at hand, you can do this.
