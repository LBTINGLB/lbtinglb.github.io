ParaView Catalyst

  • ParaView Catalyst is a library that adds ParaView Analysis
  • Simulations often save their states infrequently to minimize storage requirements due to the limitation of IO capabilities
  • For example, simulations often save their states infrequently to minimize storage requirements.
  • Catalyst is a light-weight version of the ParaView server library that is designed to be directly embedded into parallel simulation codes to perform in situ analysis at run time
    The Catalyst User’s Guilde v2.0
    Currently only care about the building process
  • Catalyst is a subset of the ParaView code base. The difference is that paraView iwll by default have many more dependencies and thus will have a larger executable size. Catalyst is the flexible and specialized configuration of ParaView that is used to reduce the executable size by reducing dependencies.
  • steps: (1)set up an edition-> (2)extract the desired code from PV source tree into a Catalyst source tree -> (3)build Catalyst
  • (1) (2)python catalyze.py -i <edition_dir> -o <Catalyst_source_dir>
    • if you want to include multiple edition_dir: -i edition_dir1 -i edition_dir2
    • python catalyze.py -i Editions/Base/ -i Editions/Essentials/ -i Editions/Extras/ -i Editions/Rendering-Base -o ./src
    • Catalyst/test-catalyst.sh ~/Kitware/ParaView/catalyst/{src,build} --no-build --no-test --remove-dirs --input Base --input Essentials --input Extras --input Rendering-Base
  • (3) For configuring Catalyst from the desired build directory
  • proxy
    • vtkSMProxy manages VTK objects that are created on a server using the proxy pattern. The managed object is manipulated through properties. The type of object created and managed by vtkSMProxy is determined by the VTKClassName variable. The object is managed by getting the desired property from the proxy, changing it’s value and updating the server with UpdateVTKObjects().
    • vtkSMProxyObserver is defined here.

proxy VS widget

  • proxy
    • pqProxy: registered sever manager proxy(a base proxy)
    • vtkSMProxy: proxy for a VTK object(s) on a server
  • pqProxyWidget
    • pqProxyWidget represents a panel for a vtkSMProxy
    • it creates widgets for each of the properties
    • pqProxyWidget doesn’t show any widgets in the panel by default (after contructor). Use filterWidgets() or updatePanel() to show widgets matching criteria.

    1. search the “”ParaViewCore/ServerManager/SMApplication/Resources/views_and_representations.xml” file for the feature
    1. search the “/ParaViewCore/ServerManager/SMApplication/Resources/utilities.xml
      “ for corresponding proxy

Idea clarification

-Server and Client 

+ Server side: we use XML file to generate proxy then let the Client know about these proxys and generate widget there. Calling `updateVTKObjects` to let server generate corresponding vtk stuff. If we do not have XML, we can generate pqRepresentation(use vtkSMProxyManager) or pqPipelineSources(use pqObjectBuilder), and feed their  proxy to 
+ Client side: generate the GUI or do the rendering stuff.
  • vtkSMProxy VS pqProxy
    + pqProxy is a wrapper around vtkSMProxy. It's designed to facilitate the usage of Qt.
