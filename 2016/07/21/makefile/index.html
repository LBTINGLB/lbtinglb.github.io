A simle makefile tutorial

  • gcc -o *executable name* *.c* -I.
    • -I. makes sure that gcc will look inthe current dirctory (.)
    • two downfalls
      • lose the compile command if switch a new computer
      • recompile all of them everytime is time-consuming and iefficient

  • a simple makefile example:

    • files after :: make sure that the rule hellomake needs to be executed if any of those files change

    • you must have a tab before the gcc command

    • this is problematic because if we change the .h file the corresponding .c file would not be recompiled

+ `%.o: %.c $(DEPS)`: The rule says that the .o file depends upon the .c version of the file and the .h files included in the DEPS macro. 
+ `$(CC) -c -o $@ @< @(CFLAGS)`: The rule then says that to generate the .o file, make needs to compile the .c file using the compiler defined in the CC macro. The -c flag says to generate the object file, the -o $@ says to put the output of the compilation in the file named on the left side of the :, the $< is the first item in the dependencies list, and the CFLAGS macro is defined as above.
    * `$@`: left side of the `:`
    * `$^`: right side of the `:`
