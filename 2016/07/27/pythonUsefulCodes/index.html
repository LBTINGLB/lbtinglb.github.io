yield and generators

  • subroutines VS coroutines
    • subroutines: a function simply returns a single value, any work done by the funtion and stored in local variables is lost
      • return implies that the function is returning control of the execution to the point where the function was called
    • coroutines: a function able to yield a series of values.
      • yield implies that the transfer of control is temporary and voluntary, and our function expects to regain it in the future.
  • In python, “functions” with these capabilities are called generators.
  • Extremely useful when dealing with infinite sequences
    • Ex. the input would consusme all of the system’s memory, you can not use subroutine.
    • for a normal function, you have only one single entry point: the first line. generator allows you to enter at anyline you want
  • Generator:
    • it’s defined as a normal function. But whenever it needs to generate a value, it does so with the yield keyword rather than return.
    • generator is a special case of iterator
      • __next__() must be defined. To get the next value from a generator, we use the same built-in function as for iterators : next()(next() tkaes care of calling the generator’s next() method. Since a generator is a type of iterator, it can be used in a for loop)

conda

  • a cross-platefrom package and environment manager that installs, runs and updates packages and their dependencies. It allows you to easily set up and switch between environments on your local computer.

1.Managing conda

  • create an environment: conda create --name name-of-environment your-desired-libraries
  • activate an new environment: source activate name-of-environment
  • copy an environment: conda create --name new-name-of-environment --clone name-of-environment
  • delete an environment: conda remove --name name-of-environment --all
