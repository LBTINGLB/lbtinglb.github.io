Lecture 02 Bits, Bytes, and Integers

    • easy to store with b istable elements
    • realably transmitted on noisy and incaccurate wires
  • Byte = 8 bits
  • bit operator
    • and - &,or - |, xor - ^, not - ~
  • !0x41 -> 0x00 !0x00 -> 0x01 !!0x41 -> 0x0!
  • Stop at 17:31

  • Shift Operations

    • Left Shift: x<<y
      • Shift bit-vector x left y positions
    • Right Shift: x>>y
      • shift bit-vector x right by y positions
        • Logical shift: Fill with 0’s on left(unsigned int)
        • Arithmetric shift: Replicate most significant bit on left(int)
    • Undefined Behavior:
      • Shift amout < 0 or >= word size

Lecture 04 Floating Point

  • fractional binary numbers
    • 5 3/4 = 101.11
    • 2 7/16 = 10.0111
    • limitation:
      • we can only represent rational numbers
      • limit range of numbers(very small values? very large?)
  • Floating point representation
    • (-1)^s M 2^E (MSB)
    • sign bit s determins whether number is positive or negative
    • significand M normally a fractional value in range (1.0, 2.0)
    • Exponment E weights value by power of two
  • precision options:
    • sign - exp - frac
    • single precision: 1 - 8 - 23
    • double precision: 1 - 11 - 52
  • Exponent coded as a biased value: E = Exp - Bias
    • bias = 2^(k-1) - 1
  • significand coded with implied leading 1: M = 1.xxxx
    • frac = 00000…000 -> 1
    • frac = 11111…111 -> 2-e
      • slide 19:00 a perfect example
  • normalized number (when exp is not 0)
    • E = 1 - bias
  • Denominalized number( when exp is all 0)
    • E = Exp - Bias
    • infinity: exp = 111…1, frac = 000…0
      • 1.0/0.0
    • NaN: exp = 111…1, frac != 000…0
      • sqrt(-1), infinity - infinity
  • floating point:
    • rounding to nearest Even(default) 4 she 5 ru
    • rounding binary number
      • Even when least significant bit is 0
      • Half way when bits to right of rounding position = 100…2(comare the rounding bits with 100..)
  • FP Multiplication
  • Mathematical Properties of FP Add
    • commutative but not associative!
      • OverFlow and inexactness of rounding
      • (3.14 +1e10) - 1e10 =0, 3.14 + (1e10 - 1e10) = 3.14
  • Floating in C
    • double/floate -> int
      • truncates fractional part, like rounding toward zero
      • not defined
    • int -> double
      • exact conversion, as llong as int has <= 54 bit word size
    • int -> float
      • round according to rounding rule

Debugging Under Unix: gdb Tutorial

quick and dirty guidance

  • gdb executable-file-name
    • init gdb
  • run alias r
    • run the executable
  • start
    • start the file
  • backtrace
    • find out calling tree
    • use frame + *number* to swich between different tree
  • x + *memory-address*
    • examine the memory-address’s value
    • x/*number*b: how many bits you want to see
  • set conditional breaking points
    • break *function-name* alias b
    • condition 1 *variable-name == value*
    • break *filename:line-number*
    • delete + *line number*: delete breakpoint
  • disable
    • disable all the break point
  • continue alis c
    • move ahead of the break point
  • step
    • go through the method one line at a time and see if we can locate the source of the error
    • tips: when enter is pressed without typing a command, the last command is automatically used
  • next alias for n
    • continue without stepping into the function
  • list alias l
    • print out the source code around the current line
    • go to specific line: `l + line-number
  • print + *variable-name*
    • print + *variable-name = value*
      set + *variable-name = value*
      • change the variable value
  • info locals
    • print out all the local variable
    • info + *anything-you-want-to-check* ex: break, watch
  • compile with -g in gcc, which would include debug infomation
  • watch + *variable*
    • print the variable if it changes
