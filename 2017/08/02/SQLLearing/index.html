SQL - Structured Query language

  • SQL is a database computer language designed for the retrieval and management of data in a relation database.
  • RDMS(Relational Database Management Systems)

Overview

  • It’s a standard language for RDMS.
  • advantage:
    • Allow users to access data in the RDMS
    • Allow users to describe the data
    • Allow users to define the data in database and manipulate that data
    • Allow to embed within other language using SQL modules, libraries & precompilers
    • Allow users to create and drop database
    • Allow users to create view, stored procedure, functions in a database
    • Allow users to set permissions on tables, prcedures and views
  • SQL Process
    • Query Dispatcher
    • Optmization Engines
    • Classic Query Engine
    • SQL Query Engine

RDBMS

  • table: data stored in databsase objects
  • field/column: every table is broken up into smaller entities called fields
  • record/row: each individual entry that exists in a table
  • NULL

  • SQL Constraints: rules enforced on data columns

  • Data integrity

    • entity integrity: no duplicated rows
    • domain integrity: enforce valid entries for a given column(type, format or range of values)
    • referential integrity: rows can not be deleted which are used by other records
    • user-defined integrity: enforces some specifice business rules
  • Normalization
    • eleminate redundant data(same data in more than one table)
    • data dependencies make sense

Others

  • foreign key vs primary key
    • Primary Key: consists of one or more columns whose data contained within is used to uniquely identify each row in the table. You can think of the primary key as an address. If the rows in a table were mailboxes, then the primary key would be the listing of street addresses.
      • When defining a table you specify the primary key. A table has just one primary key, and its definition is mandatory.
    • Foreign Key: a set of one or more columns in a table that refers to the primary key in another table. There isn’t any special code, configurations, or table definitions you need to place to officially “designate” a foreign key.
