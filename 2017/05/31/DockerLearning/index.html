What’s Docker?

  • It’s a containerization plateform that allows organizations to solve multiple problems across a diverse set of requirements including security, governance, automation, support and certification over the entire application lifecycle.
  • It packages software into a standardized units for development, shipment and deployment. It insolates the software from the environment.
  • Docker CLI(command line interface) as the client and docker daemon as the server
  • container(CO) vs vm(VM)
    • CO is an abstraction at the app layer while VM is an abstraction for physical hardware.
    • COs share the same OS kernel and each runs as isolated processes in user space. Each VM includes a full copy of an OS.
    • COs start almost instantly while VMs can be slow to boot.
  • Docker engine is built on runC(runtime code) and containered.

What’s Docker and how to use it with python

  • Benefit
    • Escape the app dependency matrix. Package dependencies with your apps in Docker containers for portability and predictability during development, testing and deployment
    • Work with any stack. Eliminate conflicts and enhance security.
    • Better team collaboration. Get features and fixes into production faster.
  • What is it?
    • Everything required to make a piece of software run is packaged into isolated containers. Only libraries and settings required to make the software work are needed.
  • How to intall docker
  • Terminology
    • Container: running instance that encapsulates required software. It’s created from images. Container can expose ports and volumes to interact with other containers.
    • Image: basic element for every container. When you create an image every step is cached and can be reused (Copy On Write model).
    • Port: a TCP/UDP port in its original meaning. To keep things simple let’s assume that ports can be exposed to the outer world (accessible from host OS) or connected to other containers — accessible only from those containers and invisible to the outer world.
    • Volume: can be described as a shared folder. Volumes are initialized when a container is created. Volumes are designed to persist data, independent of the container’s lifecycle.
    • Registry — the server that stores Docker images. It can be compared to Github — you can pull an image from the registry to deploy it locally, and you can push locally built images to the registry.
  • simple commands:
    • docker run: a command to run a container. It only accepts absolute path
    • docker run -i -t --rm ubuntu /bin/bash
      • -t: assigns a pseudo-tty or terminal inside the new container
      • -i: make an interactive connection by grabbing the standard input of the container
      • -rm: automatically remove the container when the process exits. Otherweise the containers are not deleted. They exist until we exit from the session
      • --name <given-name>
      • -d runs the container in the background
    • docker run --name daemon -d ubuntu /bin/sh -c "while true; do echo hello world; sleep 1; done": To keep it running after the end of the session.
    • docker stop <container-name>: stop a running container
    • docker start <container-name>: start a stopped container
    • docker ps -a: list all containers
    • To remove all containers we can use the following command:docker rm -f $(docker ps -aq)
    • -p <Host port: container port>: ports mapping
    • -v <path>: volume mounting

Docker curriculum

  • docker run -it <image name> sh
    • -it flags attaches us to an interactive tty in the container which means that we can run as many commands in the container as we want.
    • exit: literally it means exit.
  • docker rm $(docker ps -a -q -f status=exited): remove all exited containers
    • -q: returns the numeric IDs
    • -f: filter output based on conditions provided
    • --rm: pass it to docker run when one off docker runs are desired

Kubernetes

  • A open-source system for automating deployment, scaling, and management of cntainerized applications

    • -P: publish all exposed ports to random ports
    • -p: specify a custom port. e.g. -p 8888:80

Docker Compose vs Docker Swarm vs Kubernetes vs Docker Cloud

  • Docker compose: Allows configuring and starting multiple docker containers. Used as a helper when you want to start several docker containers. It’s used to starting containers on the same host
  • Docker Swarm: It’s used for running and connecting containers on multiple hosts. It manages containers running on multiple hosts and does things like scaling, starting a new container when one crashes, networking containers …
  • Kubernetes: A container orchestration tool developed by Google. Kubernetes goal is very similar as that for Docker swarm.
  • Docker cloud: A paid enterprise docker service that allows you to build and run containers on cloud servers or local servers. It provides a Web UI and a central control panel to run and manage containers while providing all the docker features in a user friendly Web interface.
  • docker on mac: HyperKit windows: Hyper-v. You use Docker Toolbox on old Mac/Windows. Or you can use docker(VM not compatible, and a local host)
